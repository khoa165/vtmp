- Code review and merging pull requests
- As our repo is up on Github, we have decided that our main branch will be protected and can only be merged through PR (pull request) with at least 2 approvals. As team members get their assigned tasks through Github issues, they will branch out from the main branch to work on their features and make a pull request with a detailed description and video to demonstrate the updates when it is ready to be reviewed.
- Team members will then thoroughly review each other's code and give constructive comments. If everything is good to go, we will approve each other's code and merge all the pull requests together during our weekly meeting.

### ğŸ“• LEARNINGS FOR NEWBIES

Some tips that worked out for us and might be helpful for other newbies:

- Tutorials are helpful when you are absolute beginners but we found that learning happens best on the job, so no pressure to master a certain technology before jumping in.
- Remember why you started! Please set a clear purpose and expectations for everyone from the get-go. Be accountable for everyoneâ€™s time and efforts!
- Communication is key!!! Make sure to always update your progress with your teammates. Also make it a routine to report on what you are doing to mentors so they can potentially help you out.
- Understand the strengths of each member and donâ€™t hesitate to ask for help when you are stuck. USE YOUR TEAMMATES! For code explanations, for walking you through a new technology, or just for some emotional support while you cry your eyes out over a random bug ğŸ˜… We actually have never appreciated the 12-hour timezone difference more: someone was always available 24/7!
- ChatGPT and Stackoverflow are your friends ğŸ’…
- DID I MENTION USE YOUR TEAMMATES?
